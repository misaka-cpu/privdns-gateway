# 临时面板加固设计

## 背景与目标

当前 Telegram Bot 可临时开放 sing-box Clash API 与 Zashboard，但失败回滚、自动关闭、配置归属和备份恢复之间存在状态不一致的风险。本次按“方案 A”保留 Zashboard 原生能力，将其准确定位为临时观测/控制面板；持久配置仍只通过 Bot/CLI 管理。

成功标准：

- 面板只有在配置、服务和防火墙全部成功后才报告开启。
- 任一步失败都回滚到关闭状态，且不会遗失自动关闭保障。
- 过期定时器不能关闭新会话或删除新链接。
- 自定义 `clash_api` 配置不被项目接管、覆盖或清理。
- 备份和恢复不会持久化面板的临时公网监听、密钥和 UI 配置。
- Zashboard 使用项目固定版本与校验值；本地文件被替换后，下次开启会恢复已验证版本。
- 现有功能测试、静态检查和新增回归测试全部通过。

## 范围

### 纳入

1. 面板开启/关闭的事务与失败回滚。
2. 自动关闭计时器的并发与重试状态。
3. 项目默认配置、项目管理面板配置和用户自定义配置的归属判定。
4. 面板临时状态在备份与恢复过程中的净化。
5. Zashboard 固定来源、完整性检查和被替换后的自愈。
6. Bot、CLI 状态和文档中的准确文案。
7. 覆盖上述边界的回归测试。

### 不纳入

- 不新增反向代理、Web 服务、登录系统或新的监听端口。
- 不屏蔽 Zashboard 原生断开连接等控制能力；只明确提示其边界。
- 不重构无关 Bot 菜单、sing-box 配置生成或备份框架。
- 不自动修改检测到的用户自定义 `clash_api` 配置。

## 配置归属

把 `clash_api` 分为三类：

1. **项目默认关闭态**：控制器仅监听 `127.0.0.1:9090`，且没有面板密钥、UI 目录和 UI 下载地址。
2. **项目管理开启态**：控制器为 `0.0.0.0:9090`，UI 目录为项目目录，存在临时密钥，UI 下载地址为项目固定的 Zashboard 资源。
3. **用户自定义态**：不符合以上两种精确形态的任何配置。

只有前两类可由面板功能修改。遇到第三类时操作失败并给出明确提示，原配置与防火墙均保持不变。启动清理和备份净化也只处理项目管理开启态。

## 开启事务

开启操作串行执行：

1. 判定配置归属；自定义态立即退出。
2. 下载或验证固定版本 Zashboard。
3. 写入项目管理开启态配置并校验 sing-box 配置。
4. 重启并确认 sing-box 正常。
5. 添加并核验仅允许内网来源访问 9090 的防火墙规则。
6. 发送面板链接。
7. 记录链接与本次会话代号，启动自动关闭计时器。

步骤 3 至 7 任一步失败，都撤销已完成步骤：关闭防火墙、恢复项目默认关闭态、重启并确认服务。只有全部成功后才向用户报告开启。若发送链接失败，也必须立即关闭面板。

## 关闭与自动关闭

面板操作使用独立串行锁；会话状态另用短持有时间的状态锁保护。每次开启或重新计时都会生成新的会话代号：

- 旧计时器回调发现代号已过期时直接退出，不能操作当前链接或配置。
- 重新计时会取消旧计时器，并删除旧链接，避免遗留可点击入口。
- 关闭成功后才清除当前计时器和链接。
- 关闭失败时保留当前链接状态并重新安排短间隔重试，同时向管理员报告失败；不会伪报已关闭。
- 手动关闭、自动关闭和启动清理使用同一关闭事务与结果判断。

## Zashboard 完整性

Zashboard 的版本和归档 SHA256 从项目版本清单读取，Bot 不再独立维护另一组有效值。下载后先校验归档，再原子替换 UI 目录。

安装完成时记录受管 UI 的内容指纹；每次开启前重新计算并比对。文件缺失、版本变化或指纹不符时，重新下载并恢复固定版本。sing-box 的 `external_ui_download_url` 同样固定到该资源，避免原生升级接口切换到非项目来源。

## 备份与恢复

创建备份时，不直接打包项目管理开启态的 sing-box 配置；内存中生成净化后的关闭态内容写入归档。恢复外部或旧备份时，在配置校验和落盘前再次执行同一净化。这样即使备份是在面板开放期间产生，恢复后也只会得到本地监听且不含临时密钥/UI 地址的配置。

用户自定义 `clash_api` 不属于项目临时状态，不做猜测性清理。

## 用户可见行为

- 文案使用“临时观测/控制面板”，说明可查看流量、连接并断开连接，但持久规则与出口配置仍通过 Bot/CLI 修改。
- CLI 状态根据实际配置显示 `9090(local clash_api)` 或面板临时开放状态，不再固定写死。
- 启动兜底只有在真正关闭成功时打印成功；失败时打印原因并保留重试/告警线索。

## 验证策略

先新增会失败的回归测试，再做最小实现：

- 防火墙插入或核验失败时开启失败并回滚。
- 链接发送失败时面板被关闭。
- 关闭失败时状态和重试保障仍在。
- 旧计时器不能影响新会话，重新计时会删除旧链接。
- 自定义 `clash_api` 在开启、关闭和启动清理中保持原样。
- 备份与恢复都净化项目临时状态。
- UI 文件被篡改后会恢复固定版本，版本值与清单一致。
- CLI 状态和文档覆盖关闭态与临时开放态。

最后运行项目现有全部 Python、Shell、配置校验与 CI 对应测试，并复核差异只包含本设计范围。
